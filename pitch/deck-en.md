## Slide 3 · Solution

**Title: One idea in, a three-platform launch kit out**

Key points:
- Creator OS: the operating system for a creator's one-person company
- 7 modular Agents, like stations on an assembly line:
  Idea intake → Script Agent → Shoot log → Edit Agent → Thumbnail Agent → Caption Agent → Launch desk
- Every station works standalone, or chains into a full pipeline
- The key: each Agent's prompt is external, so creators can tune it themselves — anyone can use the template, and once you tune it, it's yours

Talk track:
> Creator OS is the operating system for a creator's one-person company. Seven Agents, like seven stations on a line. You speak an idea; the Script Agent turns it into a hook-driven talking-head script. You shoot, log it, the Edit Agent produces the cut, the Thumbnail Agent makes the image, the Caption Agent spits out captions for Rednote, TikTok, and X, and it all lands on the launch desk. Each station works alone, or you chain the whole thing. And every Agent's prompt is external — you tune it, and it learns your voice.

## Slide 5 · Differentiator

**Title: Others build tools. We build an AI that learns your account**

Key points:
- Every AI tool out there does the work, but it never knows whether this particular video will land
- Our moat is a calibration loop:
  1. Before you publish, the AI blind-predicts the video's performance into an immutable prediction log
  2. Three days after publish, it auto-pulls the real numbers and reconciles them against the prediction
  3. The gap gets written back into the scoring rubric — the AI gets sharper on this account every time
- Each creator's calibration data is a private asset nobody can take (the CLI version already runs)

Talk track:
> This is where we split from every AI tool. They do the work, but they never know if a video will land. We have a calibration loop. Before you publish, the AI blind-guesses how the video will perform, into a log it cannot edit. Three days later it pulls the real numbers and reconciles. Wherever it was wrong, it writes that back into the rubric. It gets sharper on your account every time. And that calibration data is yours — switch tools and you can't take it with you.

## Slide 7 · Business model

**Title: Agency-in-a-box — one person runs a whole MCN**

Key points:
- Three phases:
  1. Now: self-use validation, prove out the pipeline and the data (in progress)
  2. Near term: sell to talking-head and knowledge creators — a subscription "agency in a box"
  3. Long term: calibration data becomes the switching moat — the more you use it, the harder to leave
- Entry point: knowledge-focused talking-head creators (beachhead) — the group with the most pain and the most willingness to pay

Talk track:
> Three phases. Right now it's self-use validation, proving out the pipeline and the data. Near term, we make it a subscription and sell it to talking-head and knowledge creators — a boxed MCN for each of them. Long term, the calibration data is the moat: the more you use it, the harder it is to leave. First cut is knowledge-focused talking-head creators — they feel the pain most, and they'll pay most.

## Slide 8 · Roadmap

**Title: Start with talking-head, cover every format**

Key points:
- Now: knowledge talking-head (pure talking-head edit + knowledge videos with auto B-roll)
- Next: tutorials, vlogs, podcast clipping
- Reusable core: one pipeline + one calibration loop — a new format is just a new set of prompt templates
- Tech base: Claude-powered, reusing modules from two open-source projects (OpenMontage for footage retrieval / social-auto-upload for semi-auto publishing). No reinventing wheels

Talk track:
> We start with talking-head because it's the most standardized. But underneath is one pipeline plus one calibration loop — a new format is just a new set of prompt templates. Next we expand into tutorials, vlogs, podcast clipping. On tech, it's Claude-powered, and we reuse two open-source projects — one for footage retrieval, one for semi-auto publishing. No reinventing wheels.
