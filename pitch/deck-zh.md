## Page 1 · 钩子开场

**页标题：我的视频，剪辑、封面、文案，没有一样是我做的**

要点：

讲话稿：

## Page 2 · 问题

**页标题：创作者 80% 的时间，没花在创作上**

要点：
- 一条视频的真实流程：想选题 → 写稿 → 拍 → 剪 → 做封面 → 写三个平台的文案 → 挨个上传
- 真正「创作」的部分只有想选题和拍，剩下全是重复的体力活
- 一个人做，一条视频几个小时；想稳定日更，等于给自己开了个没工资的血汗工厂

讲话稿：
> 做过内容的都懂。一条视频从想法到发出去，中间要写稿、剪辑、抠封面、给小红书抖音 X 各写一套文案、再挨个上传。真正算创作的只有想选题和拍摄，剩下八成时间全是重复体力活。一个人扛，一条视频好几个小时。想日更？等于给自己开了个不发工资的血汗工厂。

## Page 3 · 方案

**页标题：一条想法进来，三平台发布包出去**

要点：
- Creator OS：内容创作者的一人公司操作系统
- 7 个模块化 Agent，像一条流水线上的工位：
  想法输入 → 脚本 Agent → 拍摄登记 → 剪辑 Agent → 封面 Agent → 文案 Agent → 发布台
- 每个工位可以单独用，也可以串成流水线
- 关键：每个 Agent 的提示词是外置的，创作者能自己调优——模板谁都能用，调完就成了你自己的

讲话稿：
> Creator OS 是内容创作者的一人公司操作系统。7 个 Agent，像流水线上的 7 个工位：口述想法进来，脚本 Agent 写成带钩子的口播稿，拍完登记，剪辑 Agent 出成片，封面 Agent 出图，文案 Agent 一键出小红书抖音 X 三套文案，最后进发布台。每个工位能单独用，也能串起来。而且每个 Agent 的提示词都是外置的，你能自己调——模板谁都能用，调完它就懂你了。

（视觉占位：7 模块流水线示意图，工位横向排列，每个工位一个图标）

## Page 5 · 差异化

**页标题：别人做工具，我们做「越用越懂你的账号」**

要点：
- 市面上的 AI 工具，帮你干活，但它永远不知道你这条视频到底会不会火
- 我们的核心壁垒是一个校准闭环：
  1. 发布前，AI 盲预测这条内容的表现，写进不可篡改的预测日志
  2. 发布后第 3 天，自动抓真实数据，和预测对账
  3. 差距写回评分标准——AI 越用越懂这个账号
- 每个创作者的校准数据，是别人拿不走的独有资产（命令行版已跑通）

讲话稿：
> 这是我们和所有 AI 工具最不一样的地方。别的工具帮你干活，但它永远不知道你这条会不会火。我们有个校准闭环：发布前 AI 先盲猜这条的表现，写进一份改不了的预测日志；发第 3 天自动抓真实数据对账；差在哪，写回评分标准。它越用越懂你这个号。这份校准数据是你独有的，换任何工具都带不走。

（视觉占位：闭环图 —— 预测 → 发布 → T+3 抓数 → 对账 → 写回，箭头成环）

## Page 6 · 真实数据

**页标题：这不是 PPT，是我自己天天在用**

要点：
- 我本人就是第一个种子用户，系统前身已经真实运营，发过真实成片
- 实测数据：
  - 视频预处理 56 秒，全程无人值守
  - 一条视频人工动手时间约 10 分钟（以前全手工要好几个小时）
  - 封面工厂 CoverLab 已上线，17 个模板，cover-factory-web.vercel.app
- 结论：从「几个小时」压到「10 分钟」，这是我自己量出来的

讲话稿：
> 这些不是设想，是我自己在用。我就是第一个用户，系统前身已经真实运营，真发过成片。数据我量过：视频预处理 56 秒，全程没人管；一条视频我人工动手大概 10 分钟，以前全手工要好几个小时。封面工厂已经上线了，17 个模板，网址在这。从几小时到 10 分钟，这个数是我自己跑出来的。

（视觉占位：CoverLab 网页截图 + 「56 秒 / 10 分钟」大字对比）

## Page 7 · 商业模式

**页标题：agency-in-a-box —— 一个人也能开一家 MCN**

要点：
- 三段走：
  1. 现在：自用验证，把流程和数据跑通（进行中）
  2. 近期：卖给口播类、知识类创作者，订阅制的「盒装 agency」
  3. 远期：校准数据成为迁移壁垒——用得越久越离不开
- 切入点：知识类口播创作者（beachhead），这群人最痛、最愿意付费

讲话稿：
> 商业上分三段。现在是自用验证，把流程和数据跑通。近期把它做成订阅制，卖给口播类、知识类的创作者——相当于给每个人配一家盒装 MCN。远期，校准数据就是护城河，用得越久越离不开。第一刀切知识类口播创作者，这群人做内容最痛，也最愿意付钱。

## Page 8 · Roadmap

**页标题：从口播出发，覆盖所有内容形态**

要点：
- 现在：知识类口播（纯口播剪辑 + 知识类自动配 B-roll 素材）
- 下一步扩展：教程、vlog、播客切条
- 底层能力复用：一套流水线 + 校准闭环，换个内容形态就是换套提示词模板
- 技术底座：Claude 驱动，复用两个开源项目模块（OpenMontage 素材检索 / social-auto-upload 半自动发布），不重复造轮子

讲话稿：
> 我们从口播起步，因为它最标准化。但底层是一套流水线加一个校准闭环，换内容形态只是换套提示词模板。接下来往教程、vlog、播客切条扩。技术上 Claude 驱动，还复用了两个开源项目——一个做素材检索，一个做半自动发布，不重复造轮子。
