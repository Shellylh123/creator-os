# Creator OS · 90 秒 Live Demo 脚本（中文版）

> 目标：90 秒内让评委看到「10 秒出货 → 全流程 → 人机协作 → 人点发布」
> 原则：少讲话多点击，每一步先动手再解释；全程 Claude 驱动
> 演示顺序：文案包单独用（10 秒出货）→ 流水线视图（讲全流程）→ B-roll 确认界面 → 发布台（自动填充 + 人点发布）

---

## 开场准备（演示前 30 秒，讲第 4 页时就打开）

- [ ] 浏览器已开好 3 个标签页：① 文案包工具 ② 流水线视图 ③ 发布台
- [ ] 网络确认能连；LLM 接口先跑一次热身，避免冷启动卡顿
- [ ] 手边备一段「已经跑好的转录稿」文本，随时能粘贴
- [ ] Plan B 素材就位（见文末）

---

## 段落一 · 文案包单独出货（0–20 秒）

**动作：**
1. 切到「文案包工具」标签页
2. 把一段现成的视频转录稿粘进输入框
3. 点「生成」

**话术（边点边说）：**
> 先看最小的一块，能单独用。这是一段视频的转录稿，我粘进去，点生成——

（等结果，约 5–10 秒出货）

> 好，出来了。小红书的标题正文标签、抖音标题、X 的金句、还有封面标题候选，一次全给你。以前这些我一个平台一个平台手写，现在 10 秒。

**关键：** 让评委亲眼看到「一次输入，三平台文案同时出」。这是最快见效的钩子。

---

## 段落二 · 流水线全景（20–45 秒）

**动作：**
1. 切到「流水线视图」标签页
2. 鼠标从左到右划过 7 个工位

**话术：**
> 刚才那个文案包，只是这条流水线的一个工位。整条线是这样：想法进来，脚本 Agent 写成带钩子的口播稿，拍完登记，剪辑 Agent 出成片,封面 Agent 出图,文案 Agent 出三平台文案,最后进发布台。
> 每个工位能单独用,也能一路串下来。而且每个 Agent 的提示词都能自己调——调完它就懂你的风格了。

**关键：** 让评委理解「模块化 + 可串联 + 可调优」,别停留在单点工具的印象。

---

## 段落三 · B-roll 确认界面（45–65 秒）

**动作：**
1. 点开剪辑 Agent 里的「知识类·自动配 B-roll」流程
2. 展示 AI 已经为口播稿的每句话检索好的候选素材
3. 手动划过几个,勾掉一个不合适的,换一个

**话术：**
> 知识类视频要配画面。剪辑 Agent 会给口播稿的每一句自动检索 B-roll 素材,像这样列出来。但我不是全盘照收——我扫一眼,这个不对,换掉,这个留下。
> 这就是我们的原则:Agent 干活,人做判断。它把 90% 的检索体力活干完,我只做最后那 10% 的把关。

**关键:** 这一段是「human judgment guides AI execution」的最直观证据,正好扣主题。

---

## 段落四 · 发布台 · 自动填充 + 人点发布（65–90 秒）

**动作：**
1. 切到「发布台」标签页
2. 点某平台的「准备发布」,展示上传页被自动填好:视频、封面、标题、正文、标签全部就位
3. 鼠标移到「发布」按钮上,停住,不点

**话术:**
> 最后一步,发布台。它自动把上传页填好——视频、封面、标题、正文、标签,全就位。我不用再手动填。
>（把鼠标停在发布按钮上）
> 但这个发布按钮,永远是人来点。AI 可以把一切准备到位,按下去的那一下,必须是人的决定。
>（点下发布,或停在这里作为结尾,视时间）
> 这就是 Creator OS。一个人,一群 Agent,一整家内容公司。

**关键:** 用「鼠标停在发布键上」这个动作收尾,把主题金句钉死。

---

## Plan B · 救场预案（网络 / LLM 慢时）

| 风险 | 症状 | 救场动作 |
|---|---|---|
| LLM 生成卡住 | 文案包点了不出结果 | 立刻切到预先跑好的结果截图/文本:「网络现场有点抖,这是我刚才跑的同一份,内容一样」,继续讲 |
| 整个网络断 | 页面全打不开 | 切到本地录好的 30 秒 demo 录屏播放,旁白照讲 |
| B-roll 检索慢 | 素材列表转圈 | 跳过实时检索,直接展示已配好的成片片段:「这是配好之后的效果」 |
| 时间超了 | 讲到 60 秒还没到发布台 | 砍掉段落三的换素材操作,直接跳发布台 —— 发布台的「人点发布」是必留的收尾 |

**铁律:** 无论怎么救场,段落四的「发布按钮永远人点」一定要演到,这是和黑客松主题绑定的记忆点,不能丢。

**演示前必做:** 每个段落都提前跑一遍并截图/录屏,存本地。真断网也能靠截图讲完全程。
